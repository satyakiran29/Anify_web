A sleek, premium, and dynamic landing page website for **Anify Widgets** – showcasing beautifully crafted widget designs for Android home screens. 

## 📖 What is Anify Web?

Anify Web is the official presentation platform for the Anify widget collection. Built with React and Vite, the website is designed with a modern glassmorphic aesthetic, smooth animations, and a fully responsive layout to elegantly display minimalist, colorful, and anime-inspired widgets.

## ✨ Key Features & Benefits

- **Premium Aesthetics**: Features a dark-themed, glassmorphic UI with dynamic hover effects and gradients that make the content pop.
- **Component-Driven Architecture**: Cleanly separated into modular sections (Header, Hero, Features, Showcase, Reviews, Download, etc.) for easy maintenance.
- **Fully Responsive**: Adapts seamlessly from desktop monitors down to mobile screens without breaking the layout.
- **High Performance**: Powered by Vite and React 19 for instantaneous hot-module replacement (HMR) during development and blazing fast load times in production.
- **Countdown Ready**: Includes a built-in pre-release countdown timer to build hype before product drops.

## 🚀 Getting Started

Follow these instructions to get a copy of the project up and running on your local machine for development and testing purposes.

### Prerequisites

You will need Node.js and npm (or yarn/pnpm) installed on your machine.
- Node.js (v18+ recommended)
- npm (v9+)

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/satyakiran29/Anify_web.git
   cd Anify_web
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Start the development server**
   ```bash
   npm run dev
   ```
   The application will be available at `http://localhost:5173`.

### Building for Production

To create a production-ready build:
```bash
npm run build
```
The compiled files will be generated in the `dist` directory.

To preview the production build locally:
```bash
npm run preview
```

## 🛠️ Project Structure

The project follows a clean architectural pattern to separate layouts from page content:

```text
src/
├── assets/             # Static images and icons
├── components/
│   ├── layout/         # Persistent wrappers (Header, Footer)
│   └── sections/       # Page segments (Hero, Features, Showcase...)
├── App.jsx             # Main application orchestrator
├── index.css           # Global styles and CSS variables
└── main.jsx            # Application entry point
```
